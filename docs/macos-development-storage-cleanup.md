# macOS Development Storage Cleanup

This guide is optional local storage cleanup for a developer workstation. It does
not change ASHE TOKUN business data, Supabase data, or repository source files
unless you intentionally run commands inside a project.

## Safe To Delete And Regenerate

These folders and caches are normally safe to remove when you need disk space.
They can be regenerated by installs or builds.

```bash
# Current project build output
rm -rf .next

# Current project dependencies, followed by reinstall
rm -rf node_modules
npm install

# npm cache entries
npm cache verify
npm cache clean --force

# Xcode build intermediates
rm -rf ~/Library/Developer/Xcode/DerivedData/*

# Homebrew download cache
brew cleanup
```

For other local clones, review each project before deleting build folders. Do not
delete another project directory just because it contains generated files.

## Requires Review Before Deletion

These locations can contain important work, large downloads, or app state. Review
contents before removing anything.

```bash
# Old cloned repositories
ls ~/Developer
ls ~/Projects

# iOS simulators and device support
xcrun simctl list devices
ls ~/Library/Developer/CoreSimulator/Devices
ls ~/Library/Developer/Xcode/iOS\ DeviceSupport

# Flutter and Dart caches
ls ~/.pub-cache
ls ~/flutter/bin/cache

# Project-specific build artifacts outside this repository
find ~ -name ".next" -type d -maxdepth 5
find ~ -name "node_modules" -type d -maxdepth 5
```

Delete only folders you can confidently regenerate or no longer need.
